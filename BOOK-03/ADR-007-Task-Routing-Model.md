# ADR-007

# Модель маршрутизации задач GES

## Контекст

При росте количества рабочих пространств возникает риск:

- потери задач;
- отсутствия владельца;
- дублирования работы.

---

# Решение

Каждая инженерная задача должна проходить через Route Card.

---

# Базовая модель


Task

↓

Route Card

↓

Workspace Owner

↓

Execution

↓

Result

↓

Repository


---

# Последствия

Положительные:

- прозрачность движения задач;
- понятная ответственность;
- возможность масштабирования.

Ограничения:

- требуется дисциплина создания WO.
