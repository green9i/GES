# ADR-001

# Использование GitHub как мастер-хранилища GES

## Статус

Active

---

# Контекст

GES требует независимого, версионируемого и прозрачного хранилища инженерных записей.

---

# Рассмотренные варианты

## Word

Отклонено.

Причина:

- слабое управление версиями;
- сложность коллективной работы.

---

## Система управления знаниями

Отложено.

Причина:

- зависимость от конкретной платформы.

---

## GitHub

Принято.

Причины:

- версионирование;
- история изменений;
- работа с Markdown;
- прозрачность структуры;
- независимость от формата представления.

---

# Решение

GitHub Repository используется как мастер-хранилище GES.

---

# Последствия

Положительные:

- единый источник истины;
- полная история изменений;
- возможность автоматизации.

Ограничения:

- требуется дисциплина работы с Git.
