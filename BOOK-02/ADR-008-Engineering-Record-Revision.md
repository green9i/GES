# ADR-008

# Revision Management of Engineering Records

## Управление ревизиями инженерных записей

---

# 1. Context

В процессе развития GES возник риск появления нескольких версий одного инженерного объекта.

Пример:


WO-TEMPLATE.md

WO-TEMPLATE-v2.md

WO-TEMPLATE-NEW.md


Такой подход приводит к:

- потере единого источника истины;
- появлению конфликтующих документов;
- невозможности определить актуальную версию;
- разрушению связности инженерных записей.

---

# 2. Problem Statement

Как обеспечить развитие инженерных объектов без создания параллельных копий?

---

# 3. Decision

GENESIS Engineering Standard использует модель ревизий.

Существующий инженерный объект развивается путем:


Existing Record

↓

Revision Update

↓

Version Increment

↓

Change History

↓

New State


---

# 4. Основной принцип

Один инженерный объект имеет:

- один идентификатор;
- одну мастер-запись;
- одну историю изменений.

---

# 5. Запрещенная практика

Создание копий одного объекта запрещается.

Не допускается:


WO-TEMPLATE.md

WO-TEMPLATE-v2.md

WO-TEMPLATE-final.md

WO-TEMPLATE-final-final.md


---

# 6. Правило изменения

Изменение существующей записи выполняется только через:

1. Анализ необходимости изменения.

2. Определение типа изменения.

3. Увеличение версии.

4. Добавление записи в историю изменений.

---

# 7. Версионирование

Используется семантическая модель:


Major.Minor


Пример:


1.0

↓

1.1

↓

1.2

↓

2.0


---

# 8. Типы изменений

## Minor Revision

Используется если:

- уточнено описание;
- исправлена ошибка;
- добавлен раздел;
- улучшена структура.

Пример:


1.0 → 1.1


---

## Major Revision

Используется если:

- изменена архитектурная логика;
- изменено назначение объекта;
- изменены обязательные правила.

Пример:


1.2 → 2.0


---

# 9. История изменений

Каждый инженерный объект должен содержать:


## Change History

| Version | Date | Change |
|---|---|---|
|1.0|YYYY-MM-DD|Initial version|
|1.1|YYYY-MM-DD|Updated structure|


---

# 10. Связь с Git Repository

Git является техническим механизмом хранения истории.

GES использует:

Engineering Record + Git History = Complete Traceability


---

#11. Ответственность
Автор объекта

Отвечает за:

содержание;
необходимость изменения;
корректность версии.
Documentation Center

Отвечает за:

оформление ревизии;
структуру файла;
историю изменений.
Chief Architect

Отвечает за:

архитектурные изменения;
Major Revision;
контроль целостности GES.
12. Последствия решения
Положительные
сохраняется единый источник истины;
история развития объекта прозрачна;
исключается дублирование;
упрощается сопровождение.
Ограничения
требуется дисциплина версионирования;
необходимо фиксировать причины изменений.
