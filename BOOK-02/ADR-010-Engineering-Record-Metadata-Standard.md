# ADR-010

# Стандарт метаданных инженерных записей GES

---

# 1. Context

В процессе развития GENESIS Engineering Standard выявлено различие в структуре инженерных записей.

Часть документов содержит:

- идентификатор;
- тип;
- версию;
- статус.

Часть документов содержит только текстовое содержание.

Отсутствие единого технического описания приводит к рискам:

- невозможность автоматической обработки документов;
- потеря информации о владельце объекта;
- сложность управления версиями;
- отсутствие единой структуры Repository.

---

# 2. Problem Statement

Как обеспечить единообразие всех инженерных записей GES независимо от их типа?

---

# 3. Decision

Все инженерные записи GES должны иметь обязательный блок метаданных Front Matter.

Единая структура:

---
id:
title:
type:
version:
status:
owner:
language:
created:
modified:


---

# 4. Обязательные поля
id

Уникальный идентификатор инженерного объекта.

Примеры:

ADR-010

ST-016

WP-001

WO-004

Правила:

не изменяется после создания;
используется для ссылок;
является постоянным идентификатором объекта.
title

Название инженерной записи.

Используется:

для понимания назначения;
для отображения в каталогах;
для поиска.
type

Тип инженерного объекта.

Допустимые значения определяются классификатором GES.

Примеры:

Architecture Decision Record

Standard

Work Protocol

Work Order

Operating Manual
version

Текущая версия объекта.

Формат:

Major.Minor

Пример:

1.0

1.1

2.0
status

Текущее состояние объекта.

Примеры:

Draft

Review

Active

Frozen

Retired
owner

Ответственный контур GES.

Примеры:

GES | 00 | Chief Architect

GES | 01 | Research Center

GES | 07 | Documentation Center
language

Язык содержания документа.

Пример:

ru-RU
created

Дата первоначального создания.

Формат:

YYYY-MM-DD
modified

Дата последнего изменения.

Обновляется при каждой Revision.


---

# 5. Правило применения

Любой новый инженерный объект создается только с полным набором метаданных.

Запрещено создание:

Документ без идентификатора

Документ без версии

Документ без владельца


---

# 6. Связь с управлением версиями

ADR-010 является обязательным основанием для применения:

ADR-008

Engineering Record Revision

+

ADR-010

Metadata Standard

=

Controlled Engineering Records


---

# 7. Связь с Repository

Метаданные позволяют:

создавать каталоги;
выполнять автоматический поиск;
строить связи между объектами;
отслеживать владельцев;
контролировать жизненный цикл.


---

# 8. Обновление существующих записей

Существующие документы не пересоздаются.

Применяется:

Existing Record

↓

Revision Update

↓

Add Metadata

↓

Increase Version

↓

Commit


---

# 9. Ответственность
GES | 00 | Chief Architect

Отвечает за:

структуру стандарта;
обязательность применения;
архитектурные изменения.
Documentation Center

Отвечает за:

проверку наличия метаданных;
корректность заполнения;
техническое оформление.
Автор записи

Отвечает за:

содержание;
корректность владельца;
актуальность информации.


---

# 10. Последствия решения
Положительные

Создается единая система инженерных объектов:

ID

+

Metadata

+

Content

+

History

Появляется возможность:

масштабирования Repository;
автоматизации;
интеграции с будущими системами.
Ограничения

Требуется дисциплина оформления новых объектов.
